## О проекте

Мрачное фэнтези, где мир уже проиграл последнюю битву. Три солдата из разгромленной армии спускаются в древнее подземелье - не потому что избраны, а потому что им больше некуда идти. Игрок управляет не героями, а Проводником: бывшим героем, пожертвовавшим собой ради того, чтобы существовать как голос-подсказка. Он видит больше, чем герои. Но приказывать не может.

Герои - субъекты со своим характером, страхами и правом на ошибку. Они могут прислушаться к совету. Могут проигнорировать. Могут поступить вопреки - и оказаться правы.

## Ключевые механики

### Автономный ИИ героев
Каждый герой принимает решения самостоятельно на основе нескольких слоёв:

- **Черты характера** (`CharacterTraits`) - постоянные, не меняются. Храбрость, трусость, эмпатия, закрытость, хладнокровие, тревожность.
- **Состояния** (`States`) - динамические: стресс, страх, усталость. Влияют на веса решений в реальном времени.
- **Память** (`Memory`) - запланировано: герой будет помнить прошлые советы игрока и их результат. Плохой совет в критической ситуации оставит след.

### Система принятия решений (`DecisionMaker`)
Математическая модель, которая рассчитывает вероятности трёх состояний:

```
Fight / Apathy / Retreat
```

Стресс и страх давят на решение через нелинейные модификаторы. Усталость усиливает апатию. Даже при максимальном доверии герой может поступить по-своему - случайность заложена в архитектуру намеренно.

```
panicMod      = Clamp(0..1) от (stress + fear) / 100
exhaustionMod = Clamp(0..1) от exhaustion / 100

Fight   = Bravery   × (1 - panicMod)   × (1 - exhaustionMod)      × Random(0.85..1.15)
Apathy  = Anxiety   × (1 + panicMod)   × (1 + exhaustionMod²)     × Random(0.85..1.15)
Retreat = Cowardice × (1 + panicMod²)  × (1 + exhaustionMod×0.5)  × Random(0.85..1.15)
```

Паника и усталость работают по-разному для каждого решения: бою они мешают, апатию и отступление усиливают. Паника гонит в отступление особенно резко (в квадрате). Усталость резко роняет героя в ступор (тоже в квадрате). После расчётов все три веса нормализуются до процентов — из них делается итоговый выбор.

### Система доверия
Числовой параметр от 0 до 100, индивидуальный для каждого из трёх героев. Хранится в `States.trust`. Задизайнена полностью, интеграция в `DecisionMaker` - следующий этап.

| Уровень | Поведение |
|---------|-----------|
| 0–30 | Почти всегда игнорирует |
| 31–60 | Иногда прислушивается |
| 61–100 | Склонен следовать, но не всегда |

Игрок не видит точных цифр — только поведение героев.

### Боевая система
Автобой с вмешательством. Три класса с модульной архитектурой компонентов:

- **Воин** (`MeleeLogic`) - атакует первого живого врага в слоте
- **Лучник** (`RangedLogic`) - выбирает случайного живого врага
- **Маг/Жрец** (`HealerLogic` + `RangedLogic`) - приоритет на лечение раненых союзников, при их отсутствии — магическая атака

## Архитектура

Перед каждым действием в бою герой задаёт себе четыре вопроса. Каждый вопрос - отдельный компонент:

> **«Кто я по характеру?»** → `CharacterTraits` — постоянные черты: храбрость, трусость, тревожность и другие. Задаются один раз через ScriptableObject, не меняются.

> **«Как я себя чувствую прямо сейчас?»** → `States` - живые параметры: стресс, страх, усталость. Меняются в ходе игры.

> **«Что сейчас происходит вокруг?»** → `Situations` - ScriptableObject с описанием текущей ситуации (засада, босс, обычный бой). Передача из `BattleManager` в `DecisionMaker` запланирована - в коде уже есть комментарий под это место.

> **«Что было в прошлый раз?»** → `Memory` - структура готова, логика записи и анализа в разработке.

Все четыре ответа поступают в **`DecisionMaker`** - он взвешивает их, добавляет случайность и выдаёт одно из трёх решений: `Fight`, `Apathy` или `Retreat`.

Решение получает **`AttackController`** - он исполняет его через нужную логику: `MeleeLogic`, `RangedLogic` или `HealerLogic`. Логики находят цель и вызывают `TakeDamage` или `Heal` на компоненте `Health`.

`BattleManager` - общий распорядитель: хранит слоты героев и врагов и раздаёт их логикам. Передача текущей `Situation` в `DecisionMaker` — следующий шаг.

### Файловая структура

| Файл | Назначение |
|------|------------|
| `DecisionMaker.cs` | Мозг героя. Расчёт весов → нормализация → выбор |
| `States.cs` | Динамические состояния: stress, fear, exhaustion, trust |
| `CharacterTraits.cs` | SO с постоянными чертами характера |
| `Memory.cs` | Система учета прошлых решений и накопления опыта |
| `Situations.cs` | SO: контекст ситуации - какое давление она создаёт на стресс, страх, усталость |
| `AttackController.cs` | Исполнитель решений в бою |
| `BattleManager.cs` | Управление слотами и фракциями |
| `Health.cs` | HP, урон, лечение, смерть |
| `MeleeLogic.cs` | Выбор цели для ближнего боя |
| `RangedLogic.cs` | Выбор случайной цели для дальнего боя |
| `HealerLogic.cs` | Поиск раненых союзников для лечения |
| `Unit.cs` | SO: фракция, класс, скорость атаки |

## Сеттинг

Мир мрачного фэнтези, доживающий последние дни. Некогда им управлял Мастер - сущность, которая вела армии людей против надвигающегося зла. Мастер исчез. Армия без направления проиграла финальный бой. Последний герой пожертвовал собой иначе — не в бою, а осознанно, став Проводником: слабым отголоском исчезнувшей силы.

Три солдата слышат его голос. Но они помнят: настоящий Мастер уже однажды бросил их.

**Визуальный стиль:** Low-poly 3D, вид строго сбоку. Тёмная палитра, мрачное освещение. References: Darkest Dungeon, Dark Souls 1.
